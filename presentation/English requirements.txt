// Unregistered users' requirements
01. The system must allow an unregistered to user visualize a map with every available event in the upcoming days
02. The system must allow an unregisterd user to register either as a registered user or an event organizer, providing the necessary details

// Registered users' requirements
03. Registered users' details include: Name, Surname, Date of Birth, Email, Address, Username, Password
04. The system must allow a registered user to login and logout
05. The system must allow a registered user to modify their credentials
06. The system must offer to a registered user the same functionalities of a non-registered user except for the registration
07. The system must allow a registered user to post a review of an event together with a numerical score (ranging from 1 to 5)
08. The system must allow a registered user to visualize the average scoring of an event
09. The system must allow a registered user to visualize public statistical graphs which get published by events' organizers
10. The system must allow a registered user to state his presence to an event that hasn't taken place yet
11. The system must allow a registered user to befriend a fellow registered user
12. The system must allow a registered user to visualize what events will be attendend by his befriended users
13. Whenever a registered user states his presence to an event, the registered users in his friendlist must receive a notification about it
14. The previously stated notification must take this form: "Hi, *username* has decided to take part to *event name* which takes place in *event place* on *event date*!"
15. The system must suggest a registered user other events where he might take part based on:
	a) Previous reviews (written by the user or by others) 
	b) Past events the user has alredy attended to (both regarding the place or the theme)
	c) Distance by his home
	d) Friendships

// Event organizers requirements
16. An event organizer may be: a private citizen, an organization, a town municipality
17. Private organizers' details include: Name, Surname, VAT number, Email, Password
18. The details of organizations and town municipalities include: association name, VAT number, Email, Password
19. The system must allow an event organizer to login and logout
20. The system must allow an event organizer to modify their credentials
21. The system must allow an event organizer to visualize the average scoring of his events as well as their weighted average based on how old a review is
22. The system must offer to an event organizer the same functionalities of a non-registered user except for the registration
23. The system must allow an event organizer to visualize public statistical graphs which get published by other event organizers
24. The system must allow an event organizer to register a new event providing its details
25. Events' details include: Name, Location, Ticket price (can be free), Date(s), Theme(s), Description (optional), Poster (optional)
26. The system must allow an event organizer to modify his events' date or location
27. Whenever an event organizer modifies an event details, the registered users which stated their presence will receive a notification via email containing the modified informations
28. The system must be able to show to an event organizer how many registered users stated their presence at his events
29. The system must be able to show to an event organizer various statistics for each event, such as:
	a) Average age of registered users who stated their presence at the event
	b) Evolution of the event's scoring in time
	c) Predicted income
	d) Total number of people who attendend the event, including non-registered people 
30. An event organizer is free to choose whether to publish the previous statistics or not
